Quick Start

1

Intelligent Build (Recommended)

Let the architect analyze and decide the best approach:

/laravel-agent:build invoice management system with PDF export
2

Create Features Directly

Build complete features with CRUD, views, API, and tests:

/laravel-agent:feature:make Products with categories and variants
3

Generate Tests

Create comprehensive Pest tests for any class:

/laravel-agent:test:make App/Services/PaymentService

Auto-Invoked Skills

12 total

Skills are automatically activated based on your conversation context. No commands needed - just describe what you need and Claude will apply the relevant expertise.

laravel-feature

Triggers: "build feature", "create feature", "make feature"

Complete feature development with CRUD operations, views, API endpoints, and tests. Creates self-contained features under app/Features/ with ServiceProvider, routes, controllers, models, migrations, policies, and Pest tests.

laravel-api

Triggers: "build api", "create endpoint", "rest api"

REST API development with versioning, OpenAPI documentation, filtering (Spatie Query Builder), pagination, rate limiting, and authentication (Sanctum/Passport).

laravel-database

Triggers: "migration", "database", "query", "N+1", "index", "schema", "eloquent", "slow query"

Database operations and optimization. Creates migrations with proper indexes, fixes N+1 queries with eager loading, optimizes slow queries, designs schemas, and sets up relationships. Includes Model::preventLazyLoading() patterns.

laravel-testing

Triggers: "test", "testing", "pest", "phpunit", "coverage", "tdd"

Comprehensive Pest PHP testing. Unit tests, feature tests, API tests, browser tests (Dusk). Includes factories, mocking (Event::fake, Queue::fake), database assertions, and TDD workflows.

laravel-auth

Triggers: "auth", "authentication", "permission", "role", "policy", "gate"

Authentication and authorization. Policies, gates, Sanctum/Passport setup, role-based permissions (Spatie/Laratrust), social login, 2FA, and password rules.

laravel-livewire

Triggers: "livewire", "reactive", "component", "wire:model"

Livewire 3 reactive components. Form handling, table patterns, modals, events, polling, loading states, file uploads, and real-time validation.

laravel-filament

Triggers: "filament", "admin panel", "admin dashboard"

Filament admin panel development. Resources, pages, widgets, relation managers, custom actions, form components, table columns, and Shield permission integration.

laravel-performance

Triggers: "slow", "optimize", "cache", "performance", "speed"

Performance optimization. Caching strategies, query optimization, Pulse setup, Octane configuration, memory optimization, and lazy loading prevention.

laravel-security

Triggers: "security", "vulnerability", "XSS", "SQL injection", "CSRF"

Security audits and fixes. OWASP Top 10 checks, security middleware, password rules, input validation, mass assignment protection, and encryption.

laravel-deploy

Triggers: "deploy", "production", "server", "hosting"

Deployment and hosting. Zero-downtime deployment, health checks, environment configuration, Forge/Vapor/Docker/Bref setup, and production optimization.

laravel-queue

Triggers: "queue", "job", "notification", "background"

Background jobs and notifications. Unique jobs, rate limiting, job batching, chaining, Horizon configuration, and multi-channel notifications (55+ channels).

laravel-websocket

Triggers: "websocket", "real-time", "Reverb", "broadcasting"

Real-time features with Laravel Reverb. Broadcasting events, presence channels, private channels, scaling, reconnection handling, and client-side integration.

How It Works

Request Flow

User Request: "/build invoice system" | v +------------------+ | Architect | Phase 1: Analyze request +------------------+ Phase 2: Scan codebase | Phase 3: Check pattern registry | Phase 4: Decide implementation type v +------------------+ | Feature Builder | Creates: Models, Controllers, Views +------------------+ Creates: Routes, Migrations, Policies | Creates: Pest Tests | +-----> Delegates API to: laravel-api-builder | v +------------------+ | Review Agent | Parallel reviewers: +------------------+ - Security (OWASP) | - Quality (SOLID/DRY) | - Laravel Best Practices v - Test Coverage +------------------+ | Complete! | Summary with next steps +------------------+

Architecture Decision Matrix

Request Type Implementation Location
CRUD + UI + API (complete business capability) Feature app/Features/<Name>/
Reusable logic, no UI (shared across features) Module app/Modules/<Name>/
Orchestrates multiple operations Service app/Services/
Single discrete operation Action app/Actions/<Domain>/
