## Environment

Installed WebSocket packages:
!`composer show laravel/reverb 2>/dev/null && echo "reverb=yes" || echo "reverb=no"`

## Task

Install and configure Laravel Reverb for real-time WebSocket communication. Set up environment variables, optional Redis scaling, and Supervisor configuration.

## Steps

1. **Install packages**:
   ```bash
   composer require laravel/reverb
   php artisan reverb:install
   npm install --save-dev laravel-echo pusher-js
   ```

2. **Configure environment** in `.env`:
   - Set `BROADCAST_CONNECTION=reverb`
   - Verify `REVERB_APP_ID`, `REVERB_APP_KEY`, `REVERB_APP_SECRET` (generated by `reverb:install`)
   - Add Vite vars: `VITE_REVERB_HOST`, `VITE_REVERB_PORT`, `VITE_REVERB_SCHEME`

3. **Setup Echo client** (frontend):
   - Create `resources/js/echo.js` with broadcaster config
   - Import Echo in `resources/js/app.js`
   - See `references/echo-setup.md` for client-side details

4. **Configure Reverb** (optional scaling):
   - If `--scaled`: enable Redis in `config/reverb.php` for multi-server deployments
   - Set `REVERB_REDIS_HOST`, `REVERB_REDIS_PORT`, `REVERB_REDIS_PASSWORD` as needed

5. **Setup Supervisor** (optional, for production, if `--with-supervisor`):
   - Create `/etc/supervisor/conf.d/reverb.conf`
   - Configure to run `php artisan reverb:start --host=0.0.0.0 --port=8080`
   - Run `supervisorctl reread && supervisorctl update`
   - See `references/supervisor-setup.md` for details

6. **Report**:
   - List installed packages, environment variables set, Reverb config details, and startup command.

## Commands Available

```bash
php artisan reverb:start                    # Start development WebSocket server
php artisan reverb:start --host=0.0.0.0    # Production-ready (all interfaces)
php artisan reverb:restart                  # Restart daemon
```
